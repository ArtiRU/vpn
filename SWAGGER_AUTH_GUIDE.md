# Как использовать авторизацию в Swagger UI

После того как вы добавили декоратор `@ApiBearerAuth('JWT-auth')` ко всем защищенным контроллерам, вы можете легко тестировать авторизованные эндпоинты прямо из Swagger UI.

## Инструкция по использованию

### 1. Откройте Swagger UI

Запустите приложение и перейдите по адресу:
```
http://localhost:3000/api
```

### 2. Получите JWT токен

Сначала нужно получить JWT токен. Есть два способа:

**Способ 1: Регистрация нового пользователя**
1. Найдите секцию **Authentication**
2. Откройте эндпоинт `POST /auth/register`
3. Нажмите "Try it out"
4. Введите данные:
```json
{
  "email": "test@example.com",
  "password": "SecureP@ss123"
}
```
5. Нажмите "Execute"
6. Скопируйте `access_token` из ответа

**Способ 2: Вход существующего пользователя**
1. Найдите секцию **Authentication**
2. Откройте эндпоинт `POST /auth/login`
3. Нажмите "Try it out"
4. Введите данные:
```json
{
  "email": "test@example.com",
  "password": "SecureP@ss123"
}
```
5. Нажмите "Execute"
6. Скопируйте `access_token` из ответа

### 3. Авторизуйтесь в Swagger

1. В правом верхнем углу страницы найдите кнопку **"Authorize"** (с иконкой замка 🔒)
2. Нажмите на нее
3. В открывшемся окне вставьте ваш `access_token` в поле **Value**
   - **НЕ добавляйте** слово "Bearer" - оно добавится автоматически
   - Просто вставьте сам токен: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
4. Нажмите **"Authorize"**
5. Нажмите **"Close"**

### 4. Используйте защищенные эндпоинты

Теперь все эндпоинты с иконкой замка 🔒 будут автоматически использовать ваш токен:

- **Users** - все эндпоинты
- **Subscriptions** - все эндпоинты
- **Servers** - все эндпоинты
- **Configs** - все эндпоинты
- **Sessions** - все эндпоинты
- **Payments** - все эндпоинты (кроме webhook)
- **Server Metrics** - все эндпоинты
- **Client API** - все эндпоинты

Просто откройте нужный эндпоинт, нажмите "Try it out", заполните параметры и нажмите "Execute".

### 5. Обновление токена

Access токен истекает через 15 минут. Когда это произойдет:

1. Используйте эндпоинт `POST /auth/refresh`
2. Вставьте ваш `refresh_token`:
```json
{
  "refresh_token": "ваш_refresh_token_здесь"
}
```
3. Получите новые `access_token` и `refresh_token`
4. Повторите шаг 3 для авторизации с новым токеном

### 6. Выход из системы (деавторизация в Swagger)

1. Нажмите кнопку **"Authorize"** снова
2. Нажмите **"Logout"**
3. Нажмите **"Close"**

## Визуальные подсказки в Swagger

- 🔓 **Открытая иконка замка** - эндпоинт публичный (не требует авторизации)
- 🔒 **Закрытая иконка замка** - эндпоинт защищен (требует JWT токен)
- ✅ **Зеленая галочка на замке** - вы авторизованы, токен активен

## Примеры использования

### Пример 1: Получить список серверов (Client API)
```
1. Авторизуйтесь (см. шаги выше)
2. Откройте GET /client/servers
3. Нажмите "Try it out"
4. Нажмите "Execute"
5. Получите список доступных VPN серверов
```

### Пример 2: Проверить статус подписки
```
1. Авторизуйтесь
2. Откройте GET /client/subscription/status
3. Нажмите "Try it out"
4. Нажмите "Execute"
5. Увидите информацию о вашей подписке
```

### Пример 3: Получить VPN конфигурацию
```
1. Авторизуйтесь
2. Откройте GET /client/config
3. Нажмите "Try it out"
4. Введите server_id (например, 1)
5. Нажмите "Execute"
6. Получите VPN конфигурацию
```

## Публичные эндпоинты (не требуют авторизации)

Следующие эндпоинты **не требуют** JWT токен:

- `POST /auth/register` - Регистрация
- `POST /auth/login` - Вход
- `POST /auth/refresh` - Обновление токена
- `POST /payments/webhook/stripe` - Stripe webhook (для Stripe)

## Коды ошибок авторизации

- **401 Unauthorized** - Токен отсутствует, истек или недействителен
  - Решение: Обновите токен через `/auth/refresh` или войдите заново
  
- **403 Forbidden** - У вас нет прав для этого действия
  - Решение: Проверьте вашу роль или активную подписку

## Советы

1. **Сохраните токены** - Access и refresh токены можно сохранить в текстовый файл для повторного использования
2. **Используйте refresh_token** - Вместо повторного входа используйте refresh token для получения нового access token
3. **Проверяйте время жизни** - Access token действует 15 минут, refresh token - 7 дней
4. **Тестируйте разные роли** - Создайте пользователей с разными ролями для тестирования доступа
